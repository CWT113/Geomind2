# 获取请求参数

## 通过ServletAPI获取

原生 Spring 中有一个 HttpServletRequest 对象，它在 SpringMVC 中可以用作请求处理器方法的形参直接使用，而通过它我们就可以直接获取到请求报文的对象。

::: tip

使用 HttpServletRequest 对象获取多个同名参数时，只能获取到第一个值，后面的值获取不到。

例如 `user/list?username=admin&password=123456&hobby=bastetball&hobby=footerball`，hobby 请求参数有多个值，但是只能获取到第一个值。

:::

```java {5,9}
@Controller
@RequestMapping("user")
public class UserController {
  @GetMapping("/list")
  public String list(HttpServletRequest request) {
    String username = request.getParameter("username");
    String password = request.getParameter("password");
    // 当 URL 中有多个同名参数时，HttpServletRequest 只能获取到第一个值
    String hobby = request.getParameter("hobby");
    System.out.println("username: " + username); // admin
    System.out.println("password: " + password); // 123456
    System.out.println("hobby: " + hobby); // basketball
    return "user";
  }
}
```



## 通过形参获取

如果把控制器方法的形参和 URL 请求中携带的参数名称设置相同，则 DispatchServlet 会自动将请求中的参数映射到控制器方法的形参参数上。

```java
@Controller
@RequestMapping("user")
public class UserController {
  @GetMapping("list")
  public String list(String username, String password) {
    System.out.println("username = " + username);
    System.out.println("password = " + password);
    return "user";
  }
}
```

>这里一直报错 【HTTP 状态 500 - 内部服务器错误】，搜了有帖子说新版本 SpringMVC 中，必须要加 `@RequestParam` 注解才可以。
>
>```java {3,4}
>@GetMapping("list")
>public String list(
>@RequestParam("username") String username, 
>@RequestParam("password") String password) {
>System.out.println("username = " + username);
>System.out.println("password = " + password);
>return "user";
>}
>```
>
>或者，在 IDEA 中找到 Settings -> Build,Execution,Deployment -> Compiler -> Java Compiler，在 Additional command line parameters 中添加 `-parameters` 属性。



## @RequestParam注解获取

`@RequestParam` 注解主要用于 接收 HTTP 请求参数，常用于 GET 或 POST 请求中提取 URL 的参数值。

常用的属性：

|    属性名    |    默认值    | 作用                                                         |
| :----------: | :----------: | ------------------------------------------------------------ |
|    value     | 方法的参数名 | 参数名，对应 URL 或表单中的参数名称                          |
|   requried   |     true     | 当前参数是否必须携带，默认必须携带                           |
| defaultValue |      无      | 若请求中没有携带某参数时，可以使用该属性设置默认值，同时，required 属性会自动变为 false |

::: tip 提示

- 当 required 属性为 true，且没有设置 defaultValue 属性时，如果没有携带该属性，请求处理方法会报 【HTTP 状态 400 - 错误的请求】；
- 当请求中 没有某个属性 或 有属性但是没有值（如 `?username=&password=123`）时，defaultValue 都会被使用；

:::

```java {6-8}
@Controller
@RequestMapping("user")
public class UserController {
  @GetMapping("list")
  public String list(
    @RequestParam(value = "username", required = true) String username, // username属性必须携带
    @RequestParam(value = "password", required = false, defaultValue = "12345") String password, // password属性可以不携带，默认值为12345
    @RequestParam(value = "hobby") String[] hobby) { // hobby属性必须携带，可以多个值同时携带
    System.out.println("username = " + username); // admin
    System.out.println("password = " + password); // 12345
    System.out.println("hobby = " + Arrays.toString(hobby)); // [a, b]
    return "user";
  }
}
```

此时，通过 `user/list?username=admin&hobby=a&hobby=b` 可以测试上面的例子。



## @RequestHeader注解获取

`@RequestHeader` 注解用于获取 HTTP 请求头中的信息，常用在控制器方法的参数上。

常用的属性：

|    属性名    | 默认值 | 作用                                                         |
| :----------: | :----: | ------------------------------------------------------------ |
|    value     |   无   | 请求头名称（必填）                                           |
|   required   |  true  | 是否必须携带该请求头，默认必须携带                           |
| defaultValue |   无   | 若请求头缺失时的默认值，设置 defaultValue 后，required 属性自动变为 false |

```java {7}
@Controller
@RequestMapping("user")
public class UserController {
  @GetMapping("list")
  public String list(
    @RequestParam("username") String username,
    @RequestHeader(value = "myToken", required = false, defaultValue = "123abc") String myToken) {
    System.out.println("username = " + username);
    System.out.println("myToken = " + myToken);
    return "user";
  }
}
```



## @CookieValue注解获取

`@CookieValue` 注解用于在控制器方法中直接获取请求中的 Cookie 值。

常用的属性：

|    属性名    | 默认值 | 作用                                                         |
| :----------: | :----: | ------------------------------------------------------------ |
|    value     |   无   | Cookie 名称（必填）                                           |
|   required   |  true  | 是否必须携带该 Cookie，默认为 true                           |
| defaultValue |   无   | 请求中不存在 Cookie 时的默认值，设置 defaultValue 后，required 属性自动变为 false |

```java {8,13}
@Controller
@RequestMapping("user")
public class UserController {

  @GetMapping("getCookie")
  public String getCookie(HttpServletRequest httpServletRequest) {
    // 获取一下session，前端控制台的Cookies中就会存储一个键为 JSESSIONID 的值
    HttpSession session = httpServletRequest.getSession();
    return "user";
  }

  @GetMapping("list")
  public String list(@CookieValue("JSESSIONID") String sessionId) { // 获取时要用 JSESSIONID
    System.out.println("sessionId = " + sessionId);
    return "user";
  }
}
```



## 通过POJO类获取

通过 POJO（普通 Java 对象）获取请求参数的方式可以让代码更整洁、参数管理更加请求，适合参数比较多的情况。

### GET 请求

::: tip 提示

- 使用 POJO 类的方式获取请求参数时，请求中参数的名称必须和 POJO 类中字段名称相同；
- POJO 类中必须有 getter 和 setter 方法；

:::

::: code-group

```java [UserController] {5}
@Controller
@RequestMapping("user")
public class UserController {
  @GetMapping("addUser")
  public String addUser(User user) { // URL中的参数名须和User类中的字段名称相同
    System.out.println("user = " + user);
    return "user";
  }
}
```

```java [User]
public class User {
  private String username;

  private Integer age;

  private List<String> hobbies;

  public User() {
  }

  public User(String username, Integer age, List<String> hobbies) {
    this.username = username;
    this.age = age;
    this.hobbies = hobbies;
  }

  public Integer getAge() {
    return age;
  }

  public void setAge(Integer age) {
    this.age = age;
  }

  public String getUsername() {
    return username;
  }

  public void setUsername(String username) {
    this.username = username;
  }

  public List<String> getHobbies() {
    return hobbies;
  }

  public void setHobbies(List<String> hobbies) {
    this.hobbies = hobbies;
  }

  @Override
  public String toString() {
    return "User{" +
      "username='" + username + '\'' +
      ", age=" + age +
      ", hobbies=" + hobbies +
      '}';
  }
}
```

:::



### POST 请求

POST 请求结合 POJO 类获取请求参数的方式有两种：

- POST 请求 + JSON 方式（推荐）；
- POST 请求时，直接在地址后拼接 POJO 类的字段；

::: code-group

```java [JSON方式] {5}
@Controller
@RequestMapping("user")
public class UserController {
  @PostMapping("addUserByJson")
  public String addUserByJson(@RequestBody User user) { // 使用@RequestBody注解，标识通过JSON格式获取请求参数
    System.out.println("user = " + user);
    return "user";
  }
}
```

```java [地址拼接] {5}
@Controller
@RequestMapping("user")
public class UserController {
  @PostMapping("postUser")
  public String postUser(User user) { // 地址后拼接类字段方式
    System.out.println("user = " + user);
    return "user";
  }
}
```

:::

::: details 解决 POST 方式结合 JSON 格式获取请求参数时，报 【HTTP 状态 415 - 不支持的媒体类型】错误。

1. 使用 POST + JSON 方式获取请求参数时，需要引入 jackson-databind 包

   ```xml
   <dependencies>
     <!-- https://mvnrepository.com/artifact/com.fasterxml.jackson.core/jackson-databind -->
     <dependency>
       <groupId>com.fasterxml.jackson.core</groupId>
       <artifactId>jackson-databind</artifactId>
       <version>2.19.0</version>
     </dependency>
   </dependencies>
   ```

2. 在配置类中，开启 webMVC 驱动：

   ```java {2}
   @Configuration
   @EnableWebMvc
   @ComponentScan("com.geomind")
   public class WebConfiguration implements WebMvcConfigurer {
   }
   ```

:::
